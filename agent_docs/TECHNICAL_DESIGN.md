# TECHNICAL_DESIGN & TECH_STACK

## 1. System Architecture

### 1.1 The Node Layer (Individual Agents / Cells)
* Virtual Context Manager: Intercepts all file reads/writes and API calls. Sandboxes the agent.
* Local DAG Ledger: A local .honeyhex/ directory for each agent storing blobs, trees, and commits.

### 1.2 The Hive-Daemon (Background Process)
* A continuously running process that listens to a Redis Pub/Sub channel.
* Tracks the HEAD of all active agents.
* Responsible for triggering Global Rebase when a critical Truth Commit is pushed to the central registry.

### 1.3 The Central Registry (Control Plane)
* The overarching tracking database. Maps relationships between Sub-Swarms.
* Handles the routing of Inter-Agent Pull Requests.

### 1.4 Data Flow: Inter-Agent PR
1. Agent A writes to Virtual FS branch/data-prep.
2. Agent A executes `hex push --target=AgentB`.
3. Central Registry creates a PullRequest object in PostgreSQL.
4. Hive-Daemon notifies Agent B.
5. Agent B halts current task (checkpoints), pulls the PR diff into memory, and runs its validation prompt.
6. Agent B executes `hex merge PR_ID` or `hex reject PR_ID --reason="..."`.

## 2. Tech Stack

### 2.1 Core Languages & Environments
* Language: Python 3.12+ (Strict typing required).
* Package Management: uv or poetry.

### 2.2 Databases & State Management
* Registry & Relational Data: PostgreSQL v18 (Stores swarm topologies, PR metadata, and agent relationships).
* High-Speed Messaging & Pub/Sub: Redis (Powers the Hive-Daemon for instant cross-agent rebase triggers).
* Local State (The Git backend): GitPython paired with local JSON/Markdown file serialization to represent the Agent DAG.

### 2.3 AI & Data Processing
* LLM Gateway: LiteLLM.
* Data Validation: Pydantic v2.
* High-Performance Data Evaluation: polars (Used by Validator Agents to rapidly evaluate large datasets generated by scraping/data agents during the CI/CD thought-tests).

### 2.4 CLI & Orchestration
* CLI Framework: Typer or Click (For exposing commands like `hex merge --quorum` and `hex checkout -b`).
* Virtual Filesystem: pyfilesystem2 (Creates the sandboxed memory environment for agents to interact with before a merge).
